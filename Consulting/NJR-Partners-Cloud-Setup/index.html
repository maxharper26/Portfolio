NJR Pilot Applications

Investment data is often fragmented across PDFs, emails, websites, and APIs, making manual collection time-consuming and error-prone. Key risk and performance parameters are spread across multiple sources, while unstructured commentary remains difficult to incorporate into dashboards and governance cycles.


I developed a lightweight, serverless system that automates these core investment-management workflows — from collecting data across emails, web endpoints, and PDFs to normalising it in SQL and piping concise, governance-ready summaries to dashboards and email. The solution uses event-driven automation with modular functions: serverless jobs ingest data on schedules or triggers, extract numeric fields via selectors while summarising text with LLMs, then persist everything to SQL and object storage before publishing to dashboards and email templates. This helps support and moniter an efficient multi-asset investment process including key risk management communications.

Data Flow

  1. Ingestion: Scheduled scrapes/API calls and email triggers detect new updates.
  2. Extraction: Numeric fields parsed with deterministic selectors; documents stored.
  3. Understanding: LLM prompts produce concise highlights (risk, drivers, changes).
  4. Validation: Compare against prior values, apply thresholds, flag anomalies.
  5. Delivery: Persist to SQL, publish to dashboards, send templated email briefings.

Impact

  • Faster reporting cycles and reduced manual collection effort.
  • Consistent, auditable parameters for risk monitoring and governance.
  • Actionable summaries that surface material changes across sources.
